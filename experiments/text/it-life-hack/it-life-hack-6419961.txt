簡単にシステムキャッシュのサイズを変更！ CacheSetで最適な仮想メモリーを設定する【PC快適術】
メモリーモジュール価格の下落と64ビット版Windows 7の普及でPCに搭載されるメモリー容量は最低で4Gバイト、最大では何と64Gバイトといった容量になってきている。一昔前では考えられない容量だ。またハイエンドのノートPCでも32Gバイトを積むモデルまである。32Gバイトと言えば登場直後のネットブックに搭載されていた内蔵SSD並みの容量だ。こうした大容量メモリー搭載のおかげでシステムの動作は快適になってきている。ただし、Windowsのシステムには相変わらず仮想メモリー機能が搭載され、たとえ大容量メモリーを搭載していたとしても仮想メモリーは利用される。


Windowsには、システムファイルをキャッシュする機能が搭載されている。よく利用するであろうシステムファイルをキャッシュ領域に置いておくことで必要なときにすぐに取り出せるようにするためだ。このキャッシュが物理メモリーを超えるサイズになるとHDD上のページングファイルに退避される。

この仮想メモリーのサイズはWindowsの設定でも決めることができるが「CacheSet」を利用すれば、リアルタイムでキャッシュ周りの設定を変更することができる。CacheSetを起動して「Working set minimum」に最小値を入力し「Working set maximum」には最大値を入力して「Apply」をクリックするだけで即座に反映される（図1、図2）。


 
図1■Windowsにも仮想メモリーのサイズを指定することができるがOSの再起動が必要ですぐには反映されない。 



 
図2■CacheSetなら「Working set minimum」に最小値を入力し「Working set maximum」に最大値を入力して「Apply」をクリックするだけだ。 


通常ではサイズを変更したらWindowsの再起動が必要だが、CacheSetはWindowsを再起動する必要がないのが楽だ。また、「Clear」をクリックすると現在のキャッシュ内のデータがすべてクリアされる。ページングファイルを再度構築させたいといったときに起動して「Clear」をクリックするといった感じで使っても重宝するだろう。
